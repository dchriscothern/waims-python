# WAIMS Python Privacy Boundary

WAIMS Python is an internal athlete-monitoring and performance-operations workspace. It is not a front-office dossier tool and it should not expose protected athlete information outside legitimate staff workflows.

## What belongs in WAIMS Python

- internal readiness, wellness, load, and force-plate data
- staff-facing availability and recovery workflow
- evidence review, data quality review, and role-based monitoring views

## What does not belong in WAIMS Python

- external prospect dossiers or portal-board evaluation workflows
- broad internal medical detail exposed to roles without a legitimate educational interest
- exports of protected student records into GM / recruiting tools

## Access rule

WAIMS Python should stay role-gated. Raw wellness and similar sensitive athlete-monitoring signals should remain restricted to the appropriate performance and medical staff roles.
